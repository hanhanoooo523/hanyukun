# WeChat 风格克隆网站 / 即时通讯 App 开发计划

## 一、项目概览

构建一个类似微信的 Web 端即时通讯应用，包含联系人、单聊/群聊、消息收发等核心功能，采用现代化技术栈与简洁 UI。

---

## 二、核心功能模块

### 2.1 用户系统
- [ ] 用户注册 / 登录（邮箱 / 手机号）
- [ ] 个人资料：头像、昵称、个性签名
- [ ] 二维码名片 / 用户搜索

### 2.2 联系人
- [ ] 好友列表（按首字母分组）
- [ ] 添加好友（搜索、扫码、推荐）
- [ ] 好友备注、分组
- [ ] 黑名单

### 2.3 聊天
- [ ] 单聊
- [ ] 群聊（创建群、邀请、群管理）
- [ ] 文本、图片、语音、表情
- [ ] 消息状态：已发送 / 已送达 / 已读
- [ ] 引用、撤回
- [ ] 聊天记录搜索

### 2.4 发现 / 动态（可选）
- [ ] 朋友圈风格动态流
- [ ] 发布文字 + 图片
- [ ] 点赞、评论

---

## 三、技术栈建议

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | React / Vue | 组件化、生态完善 |
| 状态管理 | Zustand / Pinia | 轻量、易用 |
| 实时通信 | WebSocket / Socket.io | 消息推送 |
| 后端 | Node.js (Express/Fastify) / Python (FastAPI) | REST + WebSocket |
| 数据库 | PostgreSQL + Redis | 用户与消息持久化、缓存 |
| 存储 | 云存储（如 S3 / 七牛） | 头像、图片、语音文件 |

## 四、开发阶段

### Phase 1：基础架构（约 1–2 周）
1. 项目初始化：前后端脚手架
2. 用户认证：JWT / Session
3. 数据库建模：用户、好友、会话、消息
4. WebSocket 服务与基础连接管理

### Phase 2：核心聊天（约 2–3 周）
1. 单聊界面：消息列表、输入框、发送
2. 消息同步与历史加载
3. 在线状态、打字指示
4. 群聊基础：创建群、群成员、群消息

### Phase 3：联系人（约 1 周）
1. 好友列表 UI 与接口
2. 添加好友、好友请求、备注
3. 搜索用户 / 群组

### Phase 4：富媒体（约 1–2 周）
1. 图片上传与预览
2. 表情包支持
3. 语音消息（录制、播放）【可选】
4. 文件传输【可选】

### Phase 5：体验与安全（约 1 周）
1. 消息加密（端到端可选）
2. 消息撤回、删除
3. 通知、未读数、角标
4. 响应式布局与基础性能优化

---

## 五、UI/UX 设计要点

- 左侧：会话列表（头像、昵称、最后一条消息、时间、未读数）
- 中间：聊天区域（消息气泡、时间线、已读/未读）
- 右侧：联系人 / 群资料 / 聊天设置
- 底部：输入框、附件、表情、发送
- 配色：绿色为主色，白/浅灰背景，保持简洁

---

## 六、数据结构草图

```
User: id, username, avatar, nickname, status, created_at
Friend: user_id, friend_id, remark, created_at
Conversation: id, type(single/group), members[], created_at
Message: id, conversation_id, sender_id, content, type(text/image/voice), created_at, status
Group: id, name, avatar, owner_id, members[], created_at
```

---

## 七、优先级建议

1. 先完成单聊（文本为主）
2. 再做群聊和联系人
3. 最后补充富媒体和高级功能

---

## 八、风险与注意点

- **实时性**：WebSocket 连接管理、断线重连、心跳
- **扩展性**：消息分页、虚拟滚动、后端分库分表
- **安全**：输入校验、敏感信息脱敏、防暴力破解
